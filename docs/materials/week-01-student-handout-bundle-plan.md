## 1. Purpose

This document defines exactly what students should receive before, during, and after Week 1 of French A1 and Spanish A1. It is a production plan for the student-facing bundle — not a designed handout and not a final artifact.

The Markdown source files are complete and reviewed. The next step is preparing clean distribution versions: removing internal file references, stripping any instructor-facing content, choosing formats, and establishing a delivery sequence. This plan specifies all of those decisions so the production work can proceed without re-reading every file.

This document does not modify any course files. It does not create PDFs or designed documents. It is a planning reference only.

### Spanish Ser/Estar Reference Handout — Special Note

**Do not distribute the ser/estar reference in Week 1.**

The reference (`spanish-a1/reference-ser-vs-estar.md`) is a Week 2 document. The Spanish A1 syllabus already explains this: students are told in the syllabus that ser/estar is covered in Week 2, and the Week 1 student overview correctly frames it as "Week 2 content — we'll cover it properly then." Distributing the reference in Week 1 would undermine this sequencing and send students into Session 1 trying to learn two "to be" verbs at once.

**Distribute at:** the end of Session 2 or via message before Session 3 (start of Week 2), so students arrive at Week 2 having read it once.

**Content edit required before distribution:** The final paragraph of the reference file is an instructor note and must be removed before sending to students:

> *"Instructor note: This reference supports Week 2, Sessions 3 and 4. Distribute before Session 3 so students arrive having read it..."*

Remove this paragraph entirely. Everything above it is student-appropriate.

## 7. Student-Facing Quality Checklist

Before any file is distributed to students, confirm all items below. Apply to every handout including syllabi, overviews, vocabulary files, homework sheets, and reference handouts.

| Check | Pass criteria |
|---|---|
| **Course name** | Document clearly identifies the course (e.g., "French A1 — Beginner French" or "Spanish A1 — Beginner Spanish") |
| **Language and level** | Identified on the document — not assumed |
| **Week or session number** | Clearly stated (e.g., "Week 1 Vocabulary," "Session 2 Homework") |
| **Due date or timing** | Homework files: explicit due date ("Complete before Session 2"). Overview files: "for Week 1" is sufficient. Syllabus: no due date needed. |
| **Short instructions** | Each task or section has a brief instruction in plain language — no jargon, no assumptions |
| **No instructor-only notes** | Zero instructor-facing content visible. Check for phrases like "Instructor note:", "Do not distribute before:", "administer at", "Source file:", or any notes about formative checks |
| **No draft warnings** | No "DRAFT", "v1", "work in progress", or similar markers |
| **No internal file references** | No visible Markdown file paths like `week-01-vocabulary.md` or `instructor-notes.md` — convert to natural language or remove |
| **Manageable length** | Homework: 1 page maximum. Overview: 2 pages maximum. Vocabulary: 2–3 pages. Syllabus: up to 4 pages is acceptable. |
| **Consistent footer or contact line** | Each student document should include the course name and a way to contact the instructor (email or platform message) at the bottom — this does not exist in the current Markdown files and should be added during formatting, not in the source Markdown |

## 10. Next Claude Task

The next Claude task should be creating a **student-facing handout formatting standard** in Markdown — a template that defines the consistent structure every student document follows before it is exported to PDF.

This template should specify:
- Header structure: course name, level, week/session, document type
- Footer structure: course name, instructor contact line
- Section heading style
- Table formatting conventions for vocabulary and homework answer keys
- How to handle "Core" vs. "Reference" labeling consistently across languages
- Length guidance per document type
- How to handle the internal file link fix (standard replacement phrasing)

This is not designing the PDFs yet. It is the Markdown layer that sits between the source files and the final formatted output — so that every student document is produced from the same template and looks consistent without requiring Canva or a word processor for every edit.

Once the formatting standard exists, apply it to the 10 files in the production order above (Section 9) to produce distribution-ready versions of all Week 1 student materials.
